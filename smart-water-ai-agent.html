核心挑战

工业场景对 AI 的容错率极低。我们面临的最棘手问题是**“模型幻觉”**——通用大模型在回答水压、流速等专业数据时经常“一本正经地胡说八道”。此外，如何让 AI 根据实时波动的传感器数据进行精准预测，也是巨大的技术门槛。

解决方案：ReAct 范式与工作流编排

为了解决准确性问题，我没有直接让 LLM 生成答案，而是采用了 Agent Workflow (智能体工作流) 架构。

Dify 工作流视图：展示“意图识别 -> 工具调用 -> 数据校验 -> 结果生成”的闭环。
智能体工具调用流程：从用户查询到实时数据获取

🛡️ 解决幻觉 (Anti-Hallucination)

策略： 严格限制 LLM 的职责。LLM 仅负责理解用户意图（如“查一下 A 厂的水压”），然后调用固定的 API 接口获取真实数据。
结果： 确保了所有业务数据均来自数据库实况，而非模型臆造。

🔮 实时预测 (Real-time Prediction)

策略： 训练专有的时序预测小模型，并将其封装为 Tool (工具)。Agent 在工作流中按需调用该工具，根据当前传感器读数，输出未来 2 小时的压力预警。
